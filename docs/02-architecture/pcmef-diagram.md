# Диаграмма пакетов PCMEF

![PCMEF Architecture](images/pcmef-architecture.png)

## Описание слоёв

**Presentation (Представление)** — слой пользовательского интерфейса. В проекте реализован на React 18 + TypeScript. Отвечает за отображение данных, обработку пользовательских действий и маршрутизацию между экранами. Компоненты используют Zustand для управления состоянием и Axios для HTTP-запросов к бэкенду.

**Control (Управление)** — слой контроллеров REST API. Реализован в Spring Boot (`controller/`). Принимает HTTP-запросы от фронтенда, валидирует входные данные, делегирует выполнение бизнес-логики слою Mediator и формирует HTTP-ответы с DTO.

**Mediator (Посредник)** — слой бизнес-логики. Реализован через сервисные интерфейсы (`IBookingService`, `IPaymentService`) и их имплементации. Управляет транзакциями, координирует взаимодействие между Entity и Foundation, инкапсулирует правила предметной области (лимит пассажиров, таймер бронирования, проверка статусов).

**Entity (Сущность)** — слой доменных объектов и доступа к данным. Включает JPA-сущности (`Booking`, `Flight`, `Passenger`, `Payment`, `Ticket`, `User`) и репозитории Spring Data JPA. Репозитории определяют интерфейсы для CRUD-операций и пользовательских запросов к PostgreSQL.

**Foundation (Фундамент)** — слой вспомогательных механизмов. Включает MapStruct-мапперы (`BookingMapper`, `FlightMapper`, `PaymentMapper`) для преобразования Entity ↔ DTO, конфигурацию безопасности (JWT, CORS), глобальный обработчик исключений и Flyway-миграции.

## Правила зависимостей

В архитектуре PCMEF зависимости направлены строго сверху вниз:

- **Presentation** зависит от **Control** (через HTTP/REST API).
- **Control** зависит от **Mediator** (вызывает сервисные интерфейсы).
- **Mediator** зависит от **Entity** (использует репозитории и сущности).
- **Entity** зависит от **Foundation** (мапперы, конфигурации, утилиты).

Обратные зависимости запрещены. Это обеспечивает:
- независимое тестирование каждого слоя с помощью моков;
- возможность замены реализации на уровне Foundation без изменения верхних слоёв;
- чёткое разграничение ответственности между компонентами.
