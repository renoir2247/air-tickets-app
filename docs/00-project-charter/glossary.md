# Глоссарий предметной области

| Термин | Определение |
|--------|-------------|
| Авиакасса | Веб-приложение для автоматизации продажи авиабилетов кассирами и администраторами. |
| Бронирование (Booking) | Зарезервирование мест на рейс для одного или нескольких пассажиров с возможностью последующей оплаты. |
| Рейс (Flight) | Регулярное или чартерное авиационное сообщение между двумя пунктами с фиксированным расписанием. |
| Пассажир (Passenger) | Физическое лицо, для которого оформляется бронирование и выписывается билет. |
| Билет (Ticket) | Электронный документ, подтверждающий право пассажира на перевозку по конкретному рейсу. |
| Оплата (Payment) | Финансовая транзакция, инициируемая для подтверждения бронирования и генерации билетов. |
| Вебхук (Webhook) | HTTP-уведомление от платёжного провайдера о изменении статуса транзакции. |
| JWT (JSON Web Token) | Токен доступа в формате JSON, используемый для Stateless-аутентификации пользователей. |
| Кассир (Cashier) | Пользователь системы с ролью `ROLE_USER`, осуществляющий продажу билетов. |
| Администратор (Admin) | Пользователь системы с ролью `ROLE_ADMIN`, имеющий доступ к отчётам и управлению. |
| Доступные места (Available Seats) | Количество незанятых посадочных мест на рейсе, доступных для бронирования. |
| Статус бронирования | Состояние бронирования: `PENDING` (ожидает оплаты), `CONFIRMED` (подтверждено), `CANCELLED` (отменено), `EXPIRED` (просрочено). |
| Статус рейса | Состояние рейса: `SCHEDULED` (по расписанию), `DELAYED` (задержан), `CANCELLED` (отменён), `DEPARTED` (вылетел). |
| Статус оплаты | Состояние платежа: `PENDING` (в обработке), `PAID` (оплачен), `FAILED` (ошибка), `REFUNDED` (возвращён). |
| Flyway | Инструмент для версионирования и миграции схемы базы данных. |
| MapStruct | Фреймворк для генерации мапперов между сущностями и DTO на этапе компиляции. |
| DTO (Data Transfer Object) | Объект передачи данных, используемый для обмена информацией между слоями приложения. |
| Зустанда (Zustand) | Лёгкая библиотека управления состоянием для React-приложений. |
| Зод (Zod) | Библиотека для декларативной валидации и типизации данных в TypeScript. |
| CORS (Cross-Origin Resource Sharing) | Механизм разрешения кросс-доменных HTTP-запросов между фронтендом и бэкендом. |
| CSRF (Cross-Site Request Forgery) | Тип атаки на веб-приложение, при которой отключён в связи с Stateless-архитектурой JWT. |
| JaCoCo | Инструмент для измерения покрытия кода тестами в Java-приложениях. |
| IDEF0 | Методология функционального моделирования, используемая для описания бизнес-процессов. |
| BUC (Business Use Case) | Диаграмма бизнес-прецедентов, отражающая взаимодействие стейкхолдеров с системой. |
| CJM (Customer Journey Map) | Карта пути клиента, используемая для анализа пользовательского опыта. |
| ГСБД | Глобальная система бронирования и дистрибуции авиационных услуг (GDS). |
| 152-ФЗ | Федеральный закон Российской Федерации «О персональных данных», регулирующий их обработку и защиту. |
