# Руководство администратора

Администратор — пользователь с расширенными правами доступа (роль `ROLE_ADMIN`). Администратор может:

- Просматривать отчёты о продажах по всей системе
- Управлять пользователями (просмотр списка)
- Иметь доступ ко всем бронированиям

## Учётная запись администратора по умолчанию

- Логин: `admin`
- Пароль: `password`

Администратор создаётся автоматически при запуске приложения через Flyway-миграцию `V2__add_users_table.sql`.

---

## Доступ к админ-панели

1. Откройте приложение по адресу `http://localhost`
2. Войдите с учётными данными администратора
3. Нажмите «Отчёты» в навигационном меню

---

## Отчёт о продажах

Страница отчётов содержит:

- Форму выбора периода (от / до)
- Кнопку «Сформировать отчёт"
- Сводку с показателями:
  - Общее количество бронирований за период
  - Количество подтверждённых бронирований
  - Количество ожидающих бронирований
  - Количество отменённых бронирований
  - Общая выручка
  - Детальный список всех бронирований

### Статусы бронирований

- **PENDING** (Ожидает) — бронирование создано, оплата не произведена
- **CONFIRMED** (Подтверждено) — оплата прошла успешно
- **CANCELLED** (Отменено) — бронирование отменено кассиром
- **EXPIRED** (Просрочено) — время ожидания оплаты истекло

---

## Управление через API

Администратор имеет доступ к следующим эндпоинтам:

- `GET /api/reports/sales?from=...&to=...` — отчёт о продажах

Другие эндпоинты доступны наравне с обычными пользователями.

## Безопасность

### Рекомендации

- Смените пароль администратора после первого входа
- В production-окружении используйте HTTPS
- Ограничьте доступ к портам 80 и 8080 извне
- Регулярно создавайте резервные копии базы данных

### Создание резервной копии БД

```bash
sudo docker-compose exec db pg_dump -U aviauser aviacassa > backup_$(date +%Y%m%d).sql
```

### Восстановление БД из резервной копии

```bash
cat backup.sql | sudo docker-compose exec -T db psql -U aviauser -d aviacassa
```

---

## Устранение неполадок

- **Страница не загружается** — проверьте, запущен ли `docker-compose`
- **Ошибка входа** — проверьте учётные данные admin / password
- **Отчёт не формируется** — проверьте корректность дат (формат ГГГГ-ММ-ДД)
- **Нет данных в отчёте** — убедитесь, что за выбранный период были бронирования
